# T0.5 Comments Strategy — B2English

> Estrategia de comentarios para Lessons en beta.

---

## Purpose / Alcance

### Cubre
- Comentarios para Lessons (contenido educativo)
- Evaluación de opciones embed externo
- Recomendación para beta
- UX a alto nivel
- Moderación mínima

### NO cubre
- Implementación técnica (código, componentes)
- Base de datos propia para comentarios
- Endpoints de API
- Autenticación propia para comentarios

---

## Opciones Evaluadas

### 1. Giscus (GitHub Discussions)

**Descripción:** Widget que usa GitHub Discussions como backend.

| Aspecto | Evaluación |
|---------|------------|
| **Pros** | Gratis, moderación via GitHub, markdown nativo, reactions, threading |
| **Contras** | Requiere cuenta GitHub |
| **Fricción usuario** | Media — usuarios tech tienen GitHub; otros no |
| **Moderación** | Excelente — GitHub Discussions tools |
| **Privacidad/Ads** | ✅ Sin ads, GitHub privacy policy |

---

### 2. Utterances (GitHub Issues)

**Descripción:** Widget que usa GitHub Issues como backend.

| Aspecto | Evaluación |
|---------|------------|
| **Pros** | Gratis, muy simple, markdown, repo-based |
| **Contras** | Requiere cuenta GitHub, issues pueden mezclarse |
| **Fricción usuario** | Media — igual que Giscus |
| **Moderación** | Buena — via GitHub Issues |
| **Privacidad/Ads** | ✅ Sin ads |

---

### 3. Disqus

**Descripción:** Plataforma de comentarios third-party comercial.

| Aspecto | Evaluación |
|---------|------------|
| **Pros** | Sin requisito GitHub, familiar para usuarios |
| **Contras** | Ads en plan gratis, tracking, peso JS alto |
| **Fricción usuario** | Baja — login social o email |
| **Moderación** | Buena — panel Disqus |
| **Privacidad/Ads** | ⚠️ Ads + tracking en plan gratis |

---

## Recomendación para Beta: Giscus

### Decisión
**Giscus** es la opción recomendada para beta.

### Justificación

| Criterio | Giscus cumple |
|----------|---------------|
| Cero backend | ✅ Sin API ni DB propia |
| Coste | ✅ Gratis |
| Moderación | ✅ GitHub Discussions nativo |
| Simplicidad | ✅ Script embed simple |
| Open-source coherente | ✅ GitHub-based = audiencia tech B2English |
| Sin ads/tracking | ✅ Limpio |

### Trade-off aceptado
Usuarios sin GitHub no podrán comentar. Esto es aceptable porque:
- Target inicial es developers/tech
- GitHub es estándar en comunidad técnica
- Beta = feedback cualitativo > volumen

## UX Contract (Alto Nivel)

### Ubicación
- Bloque al final de cada Lesson
- Después del CTA de práctica
- Antes del footer

### Estados

| Estado | UI |
|--------|-----|
| Cargando | Skeleton placeholder o spinner |
| Cargado | Widget Giscus visible |
| Error | Mensaje: "Comments unavailable" |
| JS bloqueado | Fallback: link a GitHub Discussion |

### Fallback sin JS
```
[Comments require JavaScript]
View discussion on GitHub →
```

## Moderación Mínima

### Reglas (publicar en repo)
1. Respeto — no insultos ni ataques personales
2. On-topic — comentarios relacionados con el contenido
3. No spam — sin links promocionales
4. Idioma — inglés o español aceptados

### Anti-spam
- GitHub requiere cuenta verificada
- Giscus hereda protecciones de GitHub Discussions
- Rate limiting nativo

### Respuesta a abuso
1. Ocultar comentario (GitHub Discussion tools)
2. Reportar usuario si reincide
3. Bloquear de repo si necesario

### Responsable
- Diego como maintainer del repo
- GitHub notifications activas

## Ruta Futura (Salida)

### Si se decide migrar a comentarios nativos

**Pasos conceptuales:**
1. Crear modelo `Comment` en backend
2. Endpoint CRUD autenticado
3. Componente React propio
4. Exportar contenido de GitHub Discussions (API disponible)
5. Importar a DB propia

**Cuándo considerar migración:**
- Volumen > 100 comentarios/mes
- Usuarios piden login sin GitHub
- Necesidad de features custom (mentions, notificaciones in-app)

**Complejidad estimada:** Media-Alta

### Por ahora
Giscus es suficiente para beta. Re-evaluar en 6 meses o cuando haya tracción.
