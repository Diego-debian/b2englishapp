# T1.2 Estrategia de Visibilidad de Streak

## Problema
El "Streak" (Racha) actual se calcula localmente en el navegador (`localStorage`) y no se sincroniza con el backend. Esto causa frustración si el usuario cambia de dispositivo o limpia caché. Además, la UI actual le da una prominencia de "métrica oficial" que no merece dada su fragilidad.

## Decisión: Ocultar Streak (Estrategia Preferida)
Para cumplir con el principio de honestidad en la UI, eliminaremos la visibilidad del Streak en la interfaz principal hasta que esté implementado en el backend.

## Cambios Aplicados

### 1. Dashboard (`/app/dashboard/page.tsx`)
- **Header**: Eliminar el texto "Racha: X días".
- **Grid Card**: Reemplazar la tarjeta de "Racha Actual" por una tarjeta de **"Experiencia Total"**.
    - La tarjeta central ahora mostrará el XP Total (dato real y persistente).
    - Mantiene el balance visual de 3 columnas.

### 2. Marketing / Auth (`/components/AuthLayout.tsx`)
- Eliminar la mención a "Fire / Streak" en los bullets de features para no prometer lo que no está activo.

### 3. Practice Page (`/app/practice/page.tsx`)
- Verificar que no se renderice. (El código tiene lógica interna `syncStreak` pero no UI visible detectada).

## Resultado
La aplicación se siente más sólida al mostrar solo métricas en las que el usuario puede confiar (Nivel, XP, Verbos Aprendidos).
