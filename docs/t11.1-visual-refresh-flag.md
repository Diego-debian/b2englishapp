# Visual Refresh (T11.1) Feature Flag

 **Feature Flag**: `NEXT_PUBLIC_FEATURE_UI_V11`
 **Default**: `false` (Legacy UI preserved)
 **Purpose**: Enables a "mini design system" to unify the visual style of the Content Feed, Detail, and Admin pages without massive refactoring.

## Affected Surfaces

When enabled (`true`):

1.  **Content Feed** (`/content/feed` - Legacy View)
    - Applies `ds.layout.page` (slate-50 background, consistent padding).
    - Applies `ds.card.root` (white cards, hover lift, consistent border).
    - Applies `ds.typo` to headers.

2.  **Content Detail** (`/content/[slug]` - Legacy View)
    - Unifies font sizes and spacing with the Feed.
    - Standardizes buttons (Share/Copy tokens).

3.  **Admin Content** (`/admin/content`)
    - Standardizes container width.
    - Standardizes Page Header typography.

## How to Test

### 1. Verification (Flag OFF)
Ensure `.env.local` has:
```bash
NEXT_PUBLIC_FEATURE_UI_V11=false
# or simply remove the line
```
- **Check**: Visit `/content/feed`. Should look **exactly** as before (black background if that was default, or existing styles).
- **Check**: Visit `/admin/content`. Should look as before.

### 2. Verification (Flag ON)
Update `.env.local`:
```bash
NEXT_PUBLIC_FEATURE_UI_V11=true
```
- **Restart**: `npm run dev`.
- **Check**: Visit `/content/feed`.
    - Background should be light (`bg-slate-50`).
    - Cards should be white with shadow.
    - Typography should feel consistent.
- **Check**: Visit `/admin/content`.
    - Header font should match the Feed's header font.
    - Container max-width should align with public pages.

## Rollback
Simply set the flag to `false` or remove it to revert to the exact previous state. All changes are wrapped in conditional logic within `lib/designSystem.ts`.
