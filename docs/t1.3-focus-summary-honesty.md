# T1.3 Estrategia de Honestidad en Focus Summary

## Principio
El resumen de la sesión ("Focus Summary") debe ser una fuente de verdad absoluta para el estudiante. No debe adornar resultados ni mostrar marcadores de posición ("Lorem ipsum") si faltan datos.

## Reglas de Renderizado

### 1. "Perfect Run"
- **Condición Estricta**: `mistakes.length === 0`.
- **Display**: Tarjeta verde de felicitación "Perfect run".
- **Honestidad**: No mostrar si hubo siquiera 1 error.

### 2. "Mistakes to Fix"
- **Condición Estricta**: `mistakes.length > 0`.
- **Display**: Lista de errores específicos.
- **Campos Requeridos**:
  - `prompt` (Pregunta)
  - `userAnswer` (Respuesta del usuario)
  - `correctAnswer` (Corrección)
  - `explanation` (Por qué)

### 3. Degradación Segura (Fallback)
Si por error de datos algún campo falta, la UI no debe romperse ni mostrar "undefined":
- Si falta `prompt` → Mostrar "Pregunta no disponible".
- Si falta `explanation` → Mostrar "Sin explicación adicional".
- Si falta `userAnswer` → Mostrar "(vacío)".

## Verificación
- Simular sesión perfecta: Debe aparecer "Perfect run".
- Simular sesión con error: Debe aparecer "Mistakes to fix".
- Simular error sin explicación: Debe aparecer el texto de fallback.
