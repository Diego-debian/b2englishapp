# T0.3 Content Spec — B2English

> Modelo de contenido para lecciones tipo CMS headless.

---

## Purpose / Alcance

### Qué es CONTENT en B2English

**Content** es el material educativo estructurado que los usuarios consumen de forma pasiva:

- Explicaciones de tiempos verbales
- Videos demostrativos
- Texto formativo
- Llamadas a la acción

Content es **consumo**, no **práctica**.

### Qué NO es Content

| Concepto | ¿Es Content? | Dónde vive |
|----------|:------------:|------------|
| Lecciones de tiempos verbales | ✅ | Este spec |
| Focus Practice (preguntas interactivas) | ❌ | `focusQuestions.ts` |
| XP / Gamificación | ❌ | `UserProgress` |
| SRS (repetición espaciada) | ❌ | `UserProgress` |
| Progress tracking | ❌ | `/progress` endpoint |

## Content Unit Principal: Lesson

### Definición Conceptual

Una **Lesson** es una unidad de contenido autocontenida que explica un tema gramatical. Cada Lesson:

- Tiene un propósito educativo claro
- Se consume de forma lineal (arriba → abajo)
- No requiere interacción activa
- Puede incluir referencias a práctica (pero no contiene práctica)

### Metadatos Mínimos

| Nombre | Significado |
|--------|-------------|
| `slug` | Identificador único URL-safe (ej: `present-simple`) |
| `title` | Título visible para el usuario |
| `description` | Resumen breve del contenido (1-2 líneas) |
| `level` | Nivel del contenido (ej: `B2`, `Beginner`, `Advanced`) |
| `status` | Estado de publicación: `draft` o `published` |

### Ejemplo Conceptual

```
Lesson: "Present Simple"
├── slug: present-simple
├── title: Present Simple
├── description: Learn when and how to use the present simple tense.
├── level: B2
├── status: published
└── blocks: [...]
```

## Block System

Las Lessons se componen de **Blocks** ordenados secuencialmente.

### Bloques Soportados en Beta

#### 1. `youtube`

**Propósito:** Embeber videos explicativos de YouTube.

| Campo | Significado |
|-------|-------------|
| `video_id` | ID del video (no URL completa) |
| `caption` | Descripción opcional del video |

**Cuándo usar:**
- Explicaciones visuales de gramática
- Ejemplos de pronunciación
- Contenido complementario de terceros

**Cuándo NO usar:**
- Videos de práctica interactiva
- Contenido no educativo
- Videos sin relación con el tema

---

#### 2. `text`

**Propósito:** Contenido textual formativo.

| Campo | Significado |
|-------|-------------|
| `content` | Texto con markdown permitido |

**Cuándo usar:**
- Explicaciones gramaticales
- Reglas y excepciones
- Ejemplos con contexto

**Cuándo NO usar:**
- Instrucciones de práctica (eso es Focus)
- Contenido extenso sin estructura
- Texto que requiere interacción

---

#### 3. `cta` (Call to Action)

**Propósito:** Invitar al usuario a una acción siguiente.

| Campo | Significado |
|-------|-------------|
| `text` | Texto del botón/enlace |
| `action` | Destino (link interno o referencia a práctica) |

**Cuándo usar:**
- Al final de una lesson para ir a práctica
- Para navegar a lesson relacionada
- Para sugerir siguiente paso

**Cuándo NO usar:**
- Múltiples CTAs seguidos
- CTAs en medio de explicaciones
- Acciones externas no educativas

---

## Reglas de Composición

### Orden Secuencial
Los blocks se renderizan en orden de aparición, de arriba hacia abajo.

```
Lesson
├── Block 1 (youtube)
├── Block 2 (text)
├── Block 3 (text)
└── Block 4 (cta)
```

### Render Lineal
Cada block se renderiza uno tras otro. No hay:
- Layouts de columnas
- Tabs o acordeones
- Condicionales de visualización

### Sin Lógica Condicional
En beta, no hay:
- Blocks que aparecen según usuario
- Contenido personalizado por nivel
- A/B testing de contenido

## Extensibilidad Futura

> **Nota:** Nuevos tipos de block pueden agregarse sin romper lessons existentes.

El sistema está diseñado para que:
- Agregar `image` block no afecta lessons con `text` y `youtube`
- Agregar `quiz` block (hipotético) no requiere migrar lessons
- Frontend ignora blocks desconocidos (graceful degradation)

**Posibles bloques futuros (NO implementar ahora):**
- `image` — imágenes ilustrativas
- `table` — tablas de conjugación
- `audio` — clips de audio
- `tip` — notas destacadas

---

## Anti-alcance (Explícito)

Lo siguiente **NO está en scope** para beta:

| Feature | Razón |
|---------|-------|
| Comentarios nativos | Complejidad de moderación |
| Gamificación en content | Separación content vs practice |
| Admin panel visual | MVP enfocado en código |
| Versionado de lessons | Complejidad prematura |
| SEO avanzado | Meta tags básicos suficientes |
| Internacionalización | Una sola langue por ahora |
| Analytics por lesson | Tracking global suficiente |

- Qué campos son requeridos vs opcionales

- Tipos TypeScript

### Frontend (sin definir cómo)

Este spec informa al frontend:
- Qué estructura esperar de una Lesson
- Qué blocks debe saber renderizar
- Qué metadata mostrar

**NO define:**
- Componentes React
- Estilos CSS
- Rutas de navegación
