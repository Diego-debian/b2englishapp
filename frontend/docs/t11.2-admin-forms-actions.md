# Task 11.2: Admin Forms & Actions Visual Refresh

## Overview
This task applies consistent visual styling to the Admin Content section, specifically focusing on forms, lists, and action buttons. All changes are guarded by the `NEXT_PUBLIC_FEATURE_UI_V11` feature flag.

## Changes

### 1. Design System Expansion (`lib/designSystem.ts`)
Added new token groups to the design system:
- **`ds.form`**: Tokens for `label`, `input`, `help`, `error`, and `row`.
- **`ds.actions`**: Token for `bar` (action button container).
- **`ds.table`**: Tokens for `th`, `td`, and `tr`.

### 2. Component Updates
- **`ContentForm.tsx`**:
  - Replaced raw styles with `ds.form.*` tokens.
  - Wrapped action buttons with `ds.actions.bar`.
- **`ContentList.tsx`**:
  - Applied `ds.table.*` tokens to the content table.
  - Used `ds.state.empty` for the empty state view.
  - Styled filter buttons and action buttons.
- **Admin Pages**:
  - `app/admin/content/new/page.tsx`: Applied `ds.layout.container` and `ds.typo.h1`.
  - `app/admin/content/[slug]/edit/page.tsx`: Applied layout, typography, and header action styling.

## Verification

### Feature Flag: `NEXT_PUBLIC_FEATURE_UI_V11`
- **Default**: `false` (OFF).
- **Control**: Toggle in `.env.local` or `lib/featureFlags.ts`.

### Verification Steps

1.  **Build Verification**:
    - Run `npm run build` to ensure no TypeScript errors or breaking changes.

2.  **Visual Check (Flag OFF)**:
    - Ensure the Admin Content section looks **exactly** as it did before.
    - Check Forms, Tables, and Lists.

3.  **Visual Check (Flag ON)**:
    - **Forms**: Labels should be semi-bold slate-700. Inputs should have focus rings (violet-500). Action bars should have a top border and spacing.
    - **Tables**: Headers should be uppercase, slate-500. Rows should hover (bg-slate-50).
    - **Lists**: Empty states should use the new dashed border design.
    - **Responsiveness**: Check mobile view for form wrapping and table scrolling.

## Risks
- **CSS Specificity**: Ensure Tailwind classes don't conflict with legacy styles (mitigated by full replacement using `ds` helper).
- **Layout Shifts**: Potential minor layout shifts when switching flags (expected behavior, but acceptable).
