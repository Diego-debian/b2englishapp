# Task 11.6: Visual Harmonization Pass

## Overview
This task establishes a unified **V11 Visual Language** across the admin and content views. It replaces ad-hoc Tailwind classes in `lib/designSystem.ts` with a centralized semantic token system (`v11`), ensuring consistency in spacing, colors, borders, and effects when `NEXT_PUBLIC_FEATURE_UI_V11` is enabled.

## Changes

### 1. `lib/designSystem.ts`
- Introduced `v11` semantic token object:
  - **Surface**: `page` (slate-50), `card` (white), `panel` (slate-100/50).
  - **Text**: `primary` (slate-900), `secondary` (slate-700), `muted` (slate-500).
  - **Border**: `default` (slate-200), `subtle` (slate-100), `focus` (violet-500).
  - **Effects**: `shadow`, `radius` (rounded-xl), `focusRing`.
- Refactored all factory functions (`ds.layout`, `ds.card`, etc.) to compose these tokens.

### 2. Consumers
- **Automatic Harmonization**: By updating the centralized `ds` factories, all consumers (`FeedClient`, `ContentDetailClient`, `ContentList`, `ContentForm`) automatically inherit the harmonized styles without requiring file changes.

## Verification

### Feature Flag: `NEXT_PUBLIC_FEATURE_UI_V11`
- **Default**: `false` (OFF).

### Checklist

1.  **Build Verification**:
    - [ ] `npm run build` passes.

2.  **Manual Verification (Flag OFF)**:

3.  **Manual Verification (Flag ON)**:

    - [ ] **Admin List**: Consistent table borders and filter styles.

## Visual Decisions
- **Background**: `bg-slate-50` (unified page bg).
- **Cards**: `bg-white`, `border-slate-200`, `rounded-xl`, `shadow-sm`.
- **Primary Action**: `bg-violet-600` hover `bg-violet-700`.
- **Focus**: `ring-2 ring-violet-500`.
